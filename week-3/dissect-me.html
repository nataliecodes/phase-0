My first blog post as a Dev Bootcamp student

October 5, 2015

If you're confused about git, Github and version control, don't worry: I'm hoping to help explain it to you in this blog post.

I only started my Dev Bootcamp program a week ago, so I've only been coding for a week. Be that as it may, I have already learned cool things about the world of "coding."

It makes a lot of sense for programmers to use version control. Git is just one type of version control -- just like how Kleenex is actually just one brand of tissue. Version control, more generally, is vital for programmers who want to edit a file simultaneously. By using GitHub, an entire team can submit their changes to a file in an organized way.

Version control is key to good programming. Not only does it allow for tracking of code, it also allows for easier collaboration on projects.

The benifits of version control can be anything from not messing up your own work, or working with other people on the same project and still keep everything organized. Git helps you keep track of changes by letting you see every time a commit is saved. If you use github to store your changes you can always go back to what you did like a checkpoint that is saved for you online.
